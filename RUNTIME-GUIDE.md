# Runtime Guide

What a user needs to install and configure on their machine to run the `vibe` CLI. For contributor / build setup, see [`DEV-GUIDE.md`](DEV-GUIDE.md).

**Update policy.** Every change that affects user-facing setup (new env var, path convention, prerequisite, auth flow) MUST update this file in the same commit. Policy pinned in [PROP-000](spec/common/PROP-000.md).

---

## 1. Supported platforms

Windows 10/11, macOS 12+, Linux (recent glibc distros). One native binary per platform, no runtime / VM dependency.

## 2. Prerequisites

### 2.1 git

System `git` must be in `PATH`. `vibe` shells out to `git` for all registry operations.

- Windows: [Git for Windows](https://git-scm.com/download/win).
- macOS: `brew install git` or Xcode command-line tools.
- Linux: your distro's `git` package.

Verify with `git --version`.

### 2.2 SSH access to the registry (typical)

The default public registry is SSH-based. You need an SSH key the registry host recognises. Quick check: `ssh -T <host>` succeeds (or fails with "shell access disabled but authentication OK") means you are ready. HTTPS or `file://` registries need no SSH.

## 3. Paths and environment

| Path / Var | Purpose | Default |
|---|---|---|
| `~/.vibevm/` | vibevm user config dir (publish token, future settings) | created on first use |
| `~/.vibevm/git.publish.token` | GitVerse public-API token for `vibe registry publish` | absent unless you place it |
| `~/.vibe/registries/` | per-machine registry clone cache | overridden by `VIBE_REGISTRY_CACHE` |
| `<project>/.vibe/cache/` | per-project package content cache | gitignored |
| `<project>/vibe.toml` | project manifest | required, committed |
| `<project>/vibe.lock` | lockfile | generated by `vibe install`, committed |
| `VIBE_REGISTRY_CACHE` | override for the registry clone cache root | unset |
| `VIBEVM_PUBLISH_TOKEN` | publish token (wins over the file) | unset |

`~/.vibevm/` on Windows resolves to `%USERPROFILE%\.vibevm\`.

## 4. First-time flow

```
vibe init --path /path/to/project
vibe install flow:wal
```

This scaffolds a project and installs a canonical flow package from the default public registry. See [`manual-tests/`](manual-tests/) for end-to-end walkthroughs against the real registry.
